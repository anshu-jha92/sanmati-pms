Sanmati Production Automation / MES API

Base URL: http://localhost:4000  ·  111 endpoints across 23 modules

Response envelope — success { "ok": true, "data": ... }, error { "ok": false, "error": { code, message } }

Auth — get a token from POST /api/v1/auth/login, then send Authorization: Bearer <token>. IoT routes use X-API-Key per machine; ERP integration routes use a shared X-API-Key.

JWT Bearer — /api/v1/* X-API-Key — /iot/v1/* X-API-Key — /integrations/v1/* Public — /iot/*, login, /health

Tip: import Sanmati-API.postman_collection.json into Postman to test every endpoint with ready-made payloads.

POST /api/v1/auth/refresh Public (no auth)
Rotate refresh token and get a new access token
Verifies the refresh token JWT and rotates it with reuse detection: if the token hash is missing or already revoked, the whole token family is revoked (reuse detection). On success the old token is revoked and a new access/refresh pair is issued. Error codes: E_REFRESH, E_REFRESH_REUSE, E_REFRESH_REVOKED, E_REFRESH_EXPIRED.
Body fields
NameTypeReq.Description
refreshTokenstringyesThe current refresh token JWT to be rotated (min length 10).
Request body (example)

}
Response (200 example)
